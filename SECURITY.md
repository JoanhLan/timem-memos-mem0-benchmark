# Security Policy

## Reporting a vulnerability

If you discover a security issue in this benchmark harness (not in TiMEM, MemOS, or Mem0 themselves), please report it privately rather than opening a public issue.

Contact: open a GitHub issue with minimal details and ask for a private channel, or reach out via the repository owner's GitHub profile.

## Secrets

- Never commit `.env` or real API keys.
- Do not include API keys, tokens, or credentials in issues, PRs, or benchmark report attachments.

This project compares third-party memory services; users are responsible for securing their own API credentials and complying with each vendor's terms of service.
